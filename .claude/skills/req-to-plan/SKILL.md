# req-to-plan

## Output

- docs/IMPLEMENTATION_PLAN.md

## Must include

- Goal / Non-goals
- Acceptance Criteria (testable checklist)
- Ordered task list
- Impacted files/modules (likely)
- Test plan (unit/integration/e2e)
- Risks & assumptions
